# Contributing to Report Generator

Thank you for your interest in contributing to Report Generator! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/ReportGenerator.git`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Install dependencies: `npm install`
5. Start development: `npm run dev`

## Development Guidelines

### Code Style
- Follow ESLint rules: `npm run lint`
- Format code with Prettier: `npm run format`
- Use TypeScript for type safety
- Keep functions small and focused

### Commit Messages
Use conventional commits:
- `feat: add new feature`
- `fix: bug fix`
- `docs: documentation update`
- `style: code style change`
- `refactor: code refactoring`
- `test: add tests`

### Pull Requests

1. Update documentation
2. Add tests if applicable
3. Run `npm run lint` and `npm run type-check`
4. Create PR with clear description
5. Link any related issues

### Testing

Before submitting a PR:
```bash
npm run lint
npm run type-check
npm run build
```

## Reporting Issues

Use GitHub Issues with:
- Clear title
- Detailed description
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

## Feature Requests

Suggest new features with:
- Clear use case
- Implementation ideas (optional)
- Related issues/discussions

## Questions?

Ask in Discussions or contact maintainers.
